01 — Executive Summary

한 줄 결론부터:
이건 만들어야 한다.

분석 결과는 명확하다. 한국 소비자의 71%가 구독/계약 통합 관리 서비스가 필요하다고 답했고, 국내 구독경제 시장은 이미 100조 원을 돌파했으며, 가정용 렌탈 시장만 10조 7천억 원에 달한다. 그런데 이 모든 것을 "한 곳에서" 관리해주는 서비스가 국내에 존재하지 않는다.

더 중요한 사실: 토스와 뱅크샐러드조차 "금융 거래"를 보여줄 뿐, "생활 책임"을 관리하지 않는다. 마이데이터 2.0이 2025년 6월 시행되었지만, 이 역시 금융자산 조회 범위를 넓혔을 뿐 약정 종료일, 렌탈 만기, 세금 캘린더, 과태료 관리는 여전히 공백이다.

해외 사례도 마찬가지다. Rocket Money(구 Truebill)는 1조 4천억 원에 인수됐고 1,000만 명이 사용하지만, 구독 해지에 특화된 서비스다. 계약 만기 관리, 세금 일정, 렌탈 종료는 다루지 않는다. 문서 기획서가 정의한 "Life Responsibility OS"라는 개념 자체가 글로벌에도 공백 영역이다.

시장은 존재한다. 수요는 검증됐다. 경쟁자는 없다.
문제는 "만들 수 있느냐"가 아니라 "얼마나 빠르게 옳게 만드느냐"다.

핵심 인사이트: "모른다"는 게 문제다

미국 데이터를 보면 사람들이 자신의 구독 지출을 실제의 절반도 안 되게 추정한다 (월 $219 지출 → $86이라 인식). 한국도 같다. 문제는 단순히 "많이 쓴다"가 아니라 "내가 무엇을 왜 얼마나 내고 있는지 모른다"는 것이다. 이 인지 공백이 서비스의 핵심 가치 제안이다.

"한국 소비자 71%가 구독 서비스를 통합적으로 관리할 수 있는 서비스를 원한다고 답했다. 이는 대만(66%), 일본(56%)보다 높은 수치다."

Bango 구독전쟁 2024 리포트 / CIO 코리아 인용 (2024.10)

이 71%는 단순 설문 숫자가 아니다. 한국인이 유독 "관리의 필요성"을 높게 인식하는 건 문화적 특성과도 연결된다. 계약 문화(약정 중심의 통신, 렌탈, 보험), 세금 고지서 우편 수령, 복잡한 공과금 체계 등 한국의 생활 인프라 자체가 "관리할 것들"이 특히 많은 구조다.

검증된 통증 포인트: 응답자 40%가 디지털 구독 관리에 압도감을 느낀다. 37%는 여러 구독 관리가 어렵다고 답했다. 그리고 이 숫자에는 렌탈, 보험, 세금, 과태료는 포함되지도 않았다. Life Responsibility OS가 타깃하는 전체 고통은 훨씬 크다.

토스·뱅크샐러드의 구조적 한계

마이데이터 2.0 시행(2025.06)으로 토스와 뱅크샐러드는 금융자산 조회 범위를 넓혔다. 하지만 마이데이터는 본질적으로 금융기관 API에 의존하는 구조다. 코웨이 렌탈 계약은 금융기관이 아니고, 자동차보험 갱신일은 마이데이터에 없으며, 재산세 납부 일정은 위택스·홈택스의 영역이다. 이 구조적 공백은 마이데이터 3.0이 와도 해결되지 않는다.

핵심 인사이트: 마이데이터는 "이미 거래된 것"을 보여주는 후행 지표다. Life Responsibility OS는 "앞으로 관리해야 할 것"을 관리하는 선행 플랫폼이다. 이 두 개는 경쟁하는 게 아니라 상호 보완적이다. 오히려 마이데이터 API를 데이터 소스 중 하나로 활용하면 된다.
04 — Whitespace Analysis

아무도 없는
세 개의 공백

기존 서비스들이 왜 이 공간에 없는지는 합리적 이유가 있다. 그리고 그 이유가 바로 진입 장벽이자 기회다.

공백 1 — "계약 레이어"가 없다
금융 앱들은 "결제"를 추적하지만 "계약"을 관리하지 않는다. 코웨이 정수기 렌탈은 카드에서 29,900원이 빠져나가는 것만 보인다. 언제 약정이 끝나는지, 중도해지 위약금이 얼마인지, 만기 후 자동연장 조건이 뭔지는 어떤 앱도 알려주지 않는다. 이걸 관리하려면 금융 데이터가 아니라 "계약 데이터"가 필요한데, 이 인프라가 없다.
공백 2 — 비금융 청구의 파편화
자동차세(위택스), 재산세(위택스), 과태료(파인), 건강보험료(공단 EDI), 전기요금(한전), 가스요금(도시가스 앱) — 이것들은 모두 다른 사이트, 다른 앱, 다른 고지서로 흩어져 있다. 마이데이터는 이 영역을 관할하지 않는다. 통합할 수 있는 공식 API도 없다. 즉, 직접 사용자에게서 정보를 받아 구조화하는 방식만이 유일한 해법이다. 이것이 진입 장벽인 동시에 모방이 어려운 해자(moat)가 된다.
공백 3 — "생활 AI 비서"의 부재
챗GPT와 클로드는 범용이고, 토스 AI는 금융 거래에 특화되어 있다. "내 정수기 약정 언제 끝나?"라고 물어볼 수 있는 AI가 없다. 사용자의 생활 맥락(내가 뭘 쓰고, 언제 갱신해야 하고, 무엇이 곧 만료되는지)을 기억하고 관리해주는 버티컬 AI 비서 영역이 완전히 비어있다.

한국이 특히
좋은 시장인 이유

단순히 수요가 많아서가 아니다. 한국의 생활 인프라 구조 자체가 이 서비스의 필요성을 구조적으로 만들어낸다.

경쟁 환경의 아이러니

토스와 뱅크샐러드가 강력하게 존재하는 것이 오히려 기회다. 이 두 서비스가 이미 "앱으로 돈 관리하는 것"을 국민에게 교육시켜 놓았다. 행동 변화 비용(behavior change cost)이 이미 지불됐다. Life Responsibility OS는 새로운 카테고리를 설득할 필요 없이, 이미 앱으로 돈 관리하는 사람들에게 "그거 말고 이것도"로 진입할 수 있다.

음성 입력 파이프라인, 이메일·문자 자동 파싱(개인정보 이슈), 공공기관 세금 자동 조회 연동, 절약 AI 분석 엔진, 중복 계약 탐지 알고리즘. 이것들은 후행 단계에서 개발하는 것이 맞다.
07 — Feature Gap Analysis

현재 구현체 기준
기능 갭 완전 분석

제공된 HTML 앱과 기획 문서를 교차 분석한 결과다. SaaS 제품으로 출시하려면 어떤 기능이 추가되어야 하는지 우선순위별로 정리했다.

🔴 P0 — 출시 전 필수 (SaaS 기본 인프라)

인증 & 사용자 계정 시스템
현재 앱은 단일 브라우저 로컬 상태. SaaS는 이메일/소셜 로그인, JWT 인증, 멀티디바이스 동기화가 필수다. Supabase Auth 또는 Firebase Auth로 2~3주면 구현 가능.
서버사이드 데이터 저장 & 동기화
현재 localStorage 기반. PostgreSQL(Supabase) 또는 Firebase Firestore로 클라우드 동기화 전환 필요. 모바일 앱 출시 시 특히 중요.
결제 시스템 연동
국내: 토스페이먼츠 또는 아임포트(나이스페이먼츠). 구독형 과금 처리, 무료체험 → 유료 전환 플로우 구현 필요.

🟠 P1 — 출시 후 3개월 내 (핵심 차별화)

채팅형 AI 입력 (서비스 정체성의 핵심)
기획서의 "사용자는 정보를 던진다, AI가 구조화한다"는 철학의 구현체. "SKT인데 월 55,000원이고 내년 8월에 약정 끝나" → AI가 파싱 → "선택약정 할인 중인가요? 가족결합 있나요?" → 자동 등록. Claude API 기반으로 구현 가능. 이게 없으면 그냥 스프레드시트다.
문서/영수증 OCR 파싱
계약서·고지서 사진 업로드 → 자동 등록. Google Cloud Vision 또는 Claude Vision API 활용. 한국어 문서 특화 프롬프트 튜닝 필요. 이 기능 하나가 "아 이거 진짜 편하다" 순간을 만든다.
카카오 알림톡 / 앱 푸시 알림
만료 30일 전, 7일 전, 1일 전 알림. 세금 납부 월 알림. 미납 경고. 앱 푸시보다 카카오 알림톡이 한국에선 훨씬 효과적 (오픈율 차이 큼). 카카오 비즈메시지 API 연동 필요.

🟡 P2 — 3~6개월 (확장 기능)

과태료 / 차량 / 주거 / 사업자 카테고리
기획서에 명시된 미구현 카테고리들. 과태료·범칙금은 파인(rfine.go.kr) 조회 연동 가능성 검토. 차량 정기검사는 자동차 등록번호 기반 알림 API 존재. 주거(월세, 전세대출, 관리비), 사업자(도메인, 클라우드, SaaS) 카테고리 추가.
미납 위험 트래킹 & 우선순위 알림
납부일이 지난 항목 자동 감지. 가산금 부과 전 긴급 알림. "이번 달 내야 할 것들" 대시보드 위젯. 현재 앱은 D-day만 보여주는데, "연체 위험" 개념이 없다.
가족/공동 관리 기능
부부 공동 계정, 명의자별 항목 분리, 공유 대시보드. 한국 가구 구조상 부부가 함께 사용하는 니즈가 강하다. 토스도 가족 연결 기능이 있어 검증된 니즈.

🟢 P3 — 6개월 이후 (프리미엄 차별화)

AI 절약 분석 & 리포트
"인터넷 재약정 시 월 11,000원 절약 가능", "미사용 구독 3개 발견 — 해지 시 월 23,500원 절감", "자동차보험 갱신 전 비교 견적" 등. AI 추천 엔진. 제휴 수익 모델과 연결 가능.
전자고지 자동 파싱
카카오페이 청구서, 네이버 전자문서, 우편물 디지털화 서비스 연동. 사용자가 아무것도 입력 안 해도 고지서가 자동으로 등록되는 경험. 궁극의 UX 목표.
사업자 전용 대시보드
개인사업자·소상공인·프리랜서 타깃. 서버, 도메인, 클라우드, SaaS, 사무실 임대료, PG 수수료 관리. 세금계산서 연동. B2B SaaS 확장의 교두보.

09 — Risk Analysis

솔직한
리스크 분석

높음

토스의 기능 확장 리스크

토스는 현재 마이데이터 2.0 적용에 집중 중이다. 만약 토스가 약정 관리, 렌탈 만기 알림 기능을 추가하기로 결정하면? 토스의 2,494만 MAU 앞에서 싸우기는 어렵다. 대응 전략: 토스는 금융 거래 기반 접근이라 수동 계약 입력, AI 대화형 파싱, 문서 보관 등 "비금융 레이어"를 단기에 구현하기 어렵다. 선점 속도와 깊이로 대응. 혹은 토스에 인수되는 것도 좋은 Exit.

높음

개인정보보호 및 규제 리스크

계약서, 보험증권, 고지서 등 민감 문서 보관은 개인정보보호법, 신용정보법 등 규제 검토 필수. 특히 금융 관련 정보 처리 시 마이데이터 사업자 인가가 필요할 수 있음. 대응 전략: 초기에는 사용자가 직접 입력한 정보만 저장하는 방식으로 규제 리스크 회피. 마이데이터 API 연동 시점에 법률 자문 필수.

중간

입력 마찰감 문제 (사용자 이탈)

모든 생활 책임을 한 곳에 모으는 것은 초기 설정 비용이 크다. "내 정수기 약정은 2026년 4월까지인데, 약정서가 어딨지?"와 같은 상황에서 사용자가 이탈할 수 있음. 대응 전략: 온보딩 AI 어시스턴트("기억나는 것부터 말해보세요")로 마찰 최소화. Quick-add 기능 (주요 서비스 템플릿 제공).

중간

수익화 타이밍 리스크

한국 소비자는 "무료 앱"에 익숙하다. 유료 전환 저항이 있을 수 있음. 대응 전략: 충분한 무료 티어 제공으로 습관 형성 후 유료 전환. "이게 없으면 불편하다"는 Aha moment 설계가 핵심.

낮음

AI OCR 정확도 이슈

계약서 OCR이 틀리게 파싱될 경우 사용자 신뢰 손상. 대응 전략: AI 파싱 결과는 항상 사용자 확인 단계를 거치게 설계. "이렇게 이해했는데 맞나요?" UI 패턴.

10 — Development Roadmap

개발자가 바로
쓸 수 있는 로드맵

현재 HTML 앱을 베이스로 SaaS로 전환하는 단계별 로드맵이다. 각 단계는 독립적으로 출시 가능한 단위로 설계했다.

Phase 0 · 현재 → 1개월
SaaS 기반 인프라 전환
스택: Next.js 14 (App Router) + Supabase (Auth + DB + Storage) + Vercel

핵심 작업: 사용자 인증 (이메일+구글 소셜 로그인), PostgreSQL 스키마 설계 (항목/카테고리/문서/알림), 기존 HTML 앱 컴포넌트 React 전환, 기본 CRUD API 엔드포인트, 클라우드 스토리지 문서 보관 (Supabase Storage).

결과물: 로그인 가능한 웹앱. 데이터가 서버에 저장됨.
Phase 1 · 1~2개월
AI 입력 엔진 구축
스택: Claude API (claude-sonnet-4-6) + Structured Output (JSON mode)

핵심 작업: 채팅형 입력 UI 및 대화 플로우, 자연어 → 항목 JSON 파싱 프롬프트 설계, 되묻기 로직 (누락 필드 감지), OCR 파이프라인 (이미지 업로드 → Claude Vision → 구조화), 파싱 결과 확인 UI ("이렇게 등록할까요?").

알림 시스템 & 모바일 앱

Phase 3 · 3~5개월

핵심 작업: 결제 시스템 (구독형, 무료체험 14일), Free/Pro/Family 티어 기능 분리, 가족 공유 기능, 과태료·주거·사업자 카테고리 추가, AI 월간 리포트 생성, 마이데이터 API 연동 (금융 구독 자동 수집 검토).

국내 어떤 서비스도 계약 만기 + 렌탈 종료 + 세금 일정 + 과태료 + 보험 + 구독을 한 곳에서 관리해주지 않는다. 글로벌에도 없다. 선점하면 카테고리 리더가 된다.
③ 돈 낼 이유가 명확하다
이 서비스가 "자동차보험 갱신일 하루 전" 알림 하나만 제대로 보내줘도 사용자는 월 4,900원을 낸다. 미국에서 Bobby, Copilot, Rocket Money 등이 월 $7~14를 받고 수백만 명이 쓰는 이유가 이것이다. "절약되는 돈 > 구독료"라는 가치 방정식이 성립한다.
④ 한국 시장이 특히 유리하다
한국은 약정 문화, 렌탈 보급률, 복잡한 세금 체계, 높은 앱 수용도가 동시에 존재하는 드문 시장이다. 그리고 토스가 "앱으로 돈 관리"라는 행동을 이미 학습시켜 놨다.
⑤ AI가 진입 장벽이 아니라 경쟁 우위다
2026년 AI 비용은 1회 입력 파싱 원가가 1~2원 수준이다. AI를 쓰는 것이 비싸지 않다. 오히려 AI 대화형 입력이 "이 앱은 뭔가 다르다"는 첫 인상을 만들어주는 핵심 차별화다.
